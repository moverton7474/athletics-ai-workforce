# ONBOARDING_FLOW.md

## Marblism-Inspired Onboarding Pattern
The onboarding flow should feel like assembling a personalized AI staff, not configuring software.

## Observed Marblism Pattern
### Step type 1: Role promise screens
Each proposed worker gets a narrative screen that explains:
- worker name
- worker role
- benefit to the organization
- 3 outcome bullets

Examples observed:
- Eva = inbox / executive assistant
- Sonny = social media manager
- Stan = lead generation / sponsors-donors-fans
- Penny = SEO blog writer
- Rachel = receptionist

### Step type 2: Signup capture
Observed sequence:
- continue with Google or email
- phone number capture for account safety

### Step type 3: Team reveal / subscription
Observed team-ready page includes:
- personalized AI team summary
- estimated hours saved per month
- full team roster
- testimonial / social proof
- trust/security/support bullets
- purchase CTA

## athletics-ai-workforce adaptation
### Recommended flow
1. Select organization type / use case
2. Enter website / context / optional documents
3. Analyze and recommend initial AI workforce
4. Show role-by-role promise screens
5. Capture signup / auth
6. Show team summary and expected outcomes
7. Choose plan + seats
8. Start workspace setup

## Design Principles
- sell outcomes before settings
- reveal the team before asking for deep configuration
- use role-specific onboarding copy
- keep setup emotionally simple and operationally clear
