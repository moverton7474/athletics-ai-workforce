# WORKER_WORKSPACE_SPEC.md

## Goal
Define the standard workspace for each AI worker.

## Core Tabs
### 1. Chat
- user-to-worker conversation
- quick actions
- status and recent work context

### 2. Outputs
- drafts
- content calendar
- generated reports
- proposals
- queue of work artifacts

### 3. Guidelines
- structured role-tuning fields
- freeform instructions
- escalation rules
- brand/department preferences

### 4. Settings
- integrations/accounts
- worker assignment mode
- channels
- schedule/automation settings
- approval rules

## Shared vs Personal
### Shared worker
- team visible
- shared outputs and context
- department-focused tasks

### Personal worker
- private context
- role/person-focused support
- limited sharing by default

## athletics-ai-workforce adaptation
Workers should feel like products, not just prompts.
Each role should have a tailored workspace surface based on its dominant output type.

### Example
- Social/Content Agent → calendar-first output view
- Executive Assistant → inbox/briefing-first view
- Sponsorship Agent → opportunities/pipeline-first view
- Compliance Agent → approvals/checklist-first view
